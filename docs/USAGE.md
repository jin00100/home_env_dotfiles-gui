# 🚀 Jin's 现代化开发环境使用教程1

欢迎使用基于 Nix Home Manager 构建的现代化开发环境！本环境将 Zsh、Zellij、Neovim 以及各种现代 Rust CLI 工具深度整合，为您提供极速、美观且统一的终端体验，并特别针对 DevOps 工作流和 SSH 远程开发进行了强化。

## 1. ⚙️ 配置生效与更新 (Nix 核心操作)

由于环境由 Nix 声明式管理，当您修改了 `~/home_env_dotfiles` 目录下的任何 `.nix` 配置文件后，**必须**运行以下命令让配置生效：

*   **`hms`** : 这是一个超级别名，等同于 `home-manager switch ...`。每次修改配置文件（如添加快捷键、安装新软件）后，只需在终端输入 `hms`，Nix 就会自动编译并应用您的最新配置。
*   **`nix-clean`** : 当您觉得 Nix 占用太多磁盘空间时，在终端输入此命令。它会自动删除旧版本的环境备份并进行垃圾回收，释放磁盘空间。
*   **`./update.sh`** : 当您想把所有已安装的软件（如 Neovim、Zellij、LSP 等）更新到最新版本时，在 `home_env_dotfiles` 目录下运行此脚本。

## 2. 🪟 终端复用器：Zellij (核心工作流)

Zellij 是替代 Tmux 的现代终端复用器。在本环境中，当您打开非 VS Code 的终端时，它会**自动启动**。

**⚠️ 最重要的快捷键：模式切换**
为了防止和 Neovim 的快捷键冲突，默认的快捷键前缀（Prefix）被改为了 **`Ctrl + g`**。
*   按下 **`Ctrl + g`**：进入 **Locked (锁定) 模式**。此时 Zellij 会忽略大部分快捷键，把所有按键原封不动传给内部程序（**使用 Neovim 时必须处于此模式**）。
*   再次按 **`Ctrl + g`**：回到 **Normal (正常) 模式**，底部状态栏会变成蓝色。

**常用快捷键 (Normal 模式下或使用 Alt 键)：**
*   `Alt + n` : 在右侧新建一个面板 (Pane)。
*   `Alt + h/j/k/l` : 在不同面板之间向 左/下/上/右 移动焦点。
*   `Alt + =` / `Alt + -` : 放大 / 缩小当前面板。
*   `Alt + i` / `Alt + o` : 切换到 上一个 / 下一个 标签页 (Tab)。
*   `Ctrl + x` : 关闭当前标签页。
*   `zj_shortcuts` : 在终端输入此命令，可以随时查看自定义的快捷键速查表。

## 3. 🌐 SSH 远程开发专属强化 [新增]

如果您经常需要 SSH 登录到远程服务器进行开发，本环境为您准备了以下神级功能：

*   **动态提示符 (Starship-SSH)**: 
    当环境侦测到您处于 SSH 会话中时，命令行左侧会**自动弹出一个粉色的 IP/Hostname 标签**。这能确保您在打开多个终端窗口时，永远清楚当前操作的是哪台服务器，防止误操作。
*   **跨设备剪贴板同步 (OSC 52)**:
    在 SSH 远程服务器上的 Neovim 里，直接按下 `y` 复制文本，利用 OSC 52 终端协议，**复制的内容会瞬间穿透 SSH 隧道，同步到您本地电脑的系统剪贴板**。您可以直接在本地的浏览器或聊天软件里 `Ctrl+V` 粘贴！

## 4. 🛠️ 现代化 CLI 工具与 DevOps 神器

传统 Linux 命令已被替换为速度更快、带语法高亮的 Rust 现代工具，并加入了处理数据的必备神器：

*   **目录切换 (`zoxide`)**: 已经全面接管并升级了 `cd` 命令。无需敲长串路径，只要您以前去过某个目录，直接输入 `cd <部分目录名>` 即可瞬间跳转（例如输入 `cd module` 就能跳到 `~/home_env_dotfiles/nix/modules`）。
*   **文件列表 (`eza`)**: `ls`, `ll`, `lt` 带有漂亮图标和 Git 状态提示。
*   **文件查看 (`bat`)**: `cat <文件>` 替代传统 cat，带行号和语法高亮。
*   **系统监控 (`btop`)**: 极高颜值、支持鼠标点击的现代化资源监控面板。
*   **数据处理 (`jq` & `yq`)**: 解析 JSON 和 YAML 的行业标准神器。
    *   用法示例：`cat pod.yaml | yq '.spec.containers'`
*   **其他工具**: `fd` (极速查找), `ripgrep / rg` (极速文本搜索)。

## 5. 📝 沉浸式编辑器：Neovim (DevOps 增强版)

在终端输入 `vi` 或 `vim` 打开深度定制的 Neovim（TokyoNight 主题）。

**DevOps 专属特性：**
*   **智能诊断 UI [新增]**: 代码报错不再是枯燥的纯文本。左侧行号区会显示彩色的 Nerd Font 警告图标（󰅚, 󰀪）。按下 `Space + e` 会弹出带有圆角边框的高端错误详情浮窗。
*   **智能 YAML/Bash/Docker 支持**: 内置专属语言服务器，提供实时缩进校验和错误提示。
*   **全套代码片段 (Snippets)**: 集成 `friendly-snippets`，快速补全大段标准代码块。

**常用快捷操作 (Space 是您的 Leader 键)：**
*   **文件导航**: `Ctrl + n` 侧边栏文件树；`-` 打开 Oil.nvim 像改文本一样改目录；`Space + f/g` 全局搜索。
*   **代码分析**: `Space + d` 跳转定义；`Space + r` 查找引用；`Space + xx` 底部显示所有项目错误。
*   **快捷浮窗**: `Ctrl + /` 弹出悬浮终端；`Space + gg` 全屏打开 LazyGit。

## 6. 🧑‍🏫 如何添加新软件或自定义配置？

Nix 的核心理念是“声明式”。**不要使用 `apt install` 或 `npm install -g`**。

**Q: 我想安装新软件 / 增加 Alias / 加环境变量，该怎么做？**
1. **加软件**: 编辑 `nix/modules/packages.nix`。
2. **加命令/变量**: 编辑 `nix/modules/zsh.nix`。
3. 修改后，在终端运行 **`hms`**。配置立即生效！

---
**💡 提示**：如果您处于 Zellij 中且发现 Neovim 快捷键突然没反应了，请检查底部状态栏，多半是您忘记按 `Ctrl + g` 切回 Locked 模式了！
